# IQS DSL 代理人宣示录 (IQS DSL Agent Manifesto)

## 0. 核心愿景 (Vision)
构建一套全量、标准、具备深度质量管理专业性的图表 DSL 语法体系，旨在支持未来“Vibe Coding”范式的系统开发与 AI 代理智能推演。本宣示录整合了系统架构规范、组件编辑逻辑、以及 QC 成果报告书的业务深度，是 IQS 系统的最高技术规范。

---

## 1. 全球语法标准 (Global Syntax Standard)

### 1.1 基础元数据 (Core Metadata)
所有组件共享的通用属性：
- `Title`: 强制性。定义图表的主旨或分析主题。
- `Color[*]`: 视觉标记。通过键值对定义特定层级的色彩。
- `Font[*]`: 排版控制。定义标题、正比或轴标签的像素大小。
- `Decimals`: 精度控制。指定统计数值的小数保留位数。

### 1.2 数据录入协议 (Data Protocol)
- **YAML-lite**: 使用 `-` 或 `[series]` 块定义的层级数据。
- **DSL Intent**: 所有输入均可被 AI 意图判定引擎触发并回填。

## 2. 组件深度规范与分析逻辑 (Component Deep Specs & Logic)

### 2.1 排列图 (Pareto Chart)
- **应用场景**: 识别主要质量问题。适用于“QC 成果 - 现状调查”或“原因确认”。
- **DSL 特性**:
  - `ShowValues: true` 显示频数标记。
  - `Color[Bar/Line/MarkLine]` 分别定义频数柱、累计百分比线和 80% 标识线。
- **深度分析逻辑**:
  - **ABC 分类法**: A类 (0-80%) 是关键少数，需立即立项；B类 (80-90%) 是次要；C类 (90-100%) 为一般。
  - **自动排序**: 系统强制执行降序排列，确保 Pareto 原则。
- **专家提示**: "解决最左侧的两个主要因素，通常能消除 80% 的质量成本。"

### 2.2 直方图 (Histogram)
- **应用场景**: 观察分布规律。适用于“现状调查”确认制程是否符合正态分布。
- **DSL 特性**:
  - `USL/LSL/Target`: 必填项，用于计算 Cp/Cpk。
  - `ShowCurve: true` 开启正态动态拟合。
- **深度分析逻辑**:
  - **工序能力指数**: Cp/Cpk ≥ 1.33 为合格（工业级门槛）；1.67 以上为优秀。
  - **模式识别**: 双峰直方图通常意味着数据混合（不同班次或原材料）。
- **专家提示**: "均值反映中心移动，标准差反映波动大小。双峰是波动的报警器。"

### 2.3 控制图 (Control Chart / SPC)
- **应用场景**: 过程稳定性监控。适用于“分析原因”或“效果检查”阶段的稳定性验证。
- **DSL 特性**:
  - `Type`: 必须精准定义（I-MR, X-bar-R, P 等）。
  - `Rules: Nelson/Western-Electric` 切换判异算法。
- **深度分析逻辑**:
  - **选型逻辑**: 
    - 计量型：小样本选 I-MR，标准样本 (2-10) 选 X-bar-R，大样本选 X-bar-S。
    - 计数型：计件选 P/NP，计点选 C/U。
  - **判异逻辑**: Nelson Rule 1 (点超出 3σ) 是核心判异点；趋势性判异 (Rule 2/3) 用于预防性预警。
- **专家提示**: "规格限 (USL/LSL) 是客户要求，控制限 (UCL/LCL) 是制程能力。两者不可混查。"

### 2.4 雷达图 (Radar / Spider Chart)
- **应用场景**: 综合评价。适用于“效果检查”阶段的多维度指标对比。
- **DSL 特性**:
  - `Standardize: true` 用于消除量纲差异（如价格与性能）。
  - `ShowAreaScore: true` 显示基于多边形面积的综合实力评分。
- **深度分析逻辑**:
  - **面积得分法**: 并非加权平均，而是侧重指标间的“木桶效应”（平衡性）。
  - **相似度检测**: 利用欧氏距离识别竞品或对象的特征重合度（>80% 为同质化竞争）。
- **专家提示**: "面积越大，表现越均衡；尖锐的夹角代表了严重的短板风险。"

### 2.5 散点图 (Scatter Plot)
- **应用场景**: 变量相关性。适用于“原因确认”验证 X 与 Y 的因果关系。
- **DSL 特性**:
  - `ShowTrend: true` 开启线性回归拟合。
  - `3D: true` 扩展至 X, Y, Z 三轴及气泡大小显示。
- **深度分析逻辑**:
  - **相关系数 (R)**: R 接近 1 或 -1 代表极强相关；0 为不相关。
  - **因果警告**: 相关≠因果。必须排除“第三变量”的干扰。
- **专家提示**: "回归线是预测模型，散点分布是波动模型。先看波动，再取趋势。"

### 2.6 鱼骨图 (Fishbone / Ishikawa)
- **应用场景**: 原因搜寻。适用于“分析原因”阶段的头脑风暴。
- **DSL 特性**:
  - 使用 `# / ## / ###` 定义大骨、中骨、小骨层级。
  - AI 模板支持 `5M1E` (制造业) 和 `4P` (服务业) 自动展开。
- **分析逻辑**: 连续追问 5 个为什么 (5 Whys)。大骨是维度，中骨是方向，小骨是末端因素。
- **专家提示**: "鱼骨图越密代表分析越深入。标记 3-5 个主骨，即为要因初步候选。"

### 2.7 矩阵图 (Matrix Diagram)
- **应用场景**: 多对多映射。适用于“对策制定”或“关联度评价”。
- **DSL 特性**:
  - `Type: L/T/Y/X/C` 定义几何布局。
  - `Weight`: 定义 ◎(9), ○(3), △(1) 的量化分值。
- **分析逻辑**:
  - L型：二元关系。
  - T/Y型：三元闭环。
  - X型：资源-岗位-风险的全维管控。
  - C型：自相关分析（质量屋顶图）。
- **专家提示**: "矩阵图是决策的量化基准。加权分值最高的交叉点，通常是控制的关键路径。"

### 2.8 关联图 (Relationship Diagram)
- **应用场景**: 复杂多重因果。适用于原因交织、无法用分层法处理的疑难问题。
- **DSL 特性**:
  - `Node` 定义节点；`Rel: A -> B` 定义流向。
- **深度分析逻辑**:
  - **出度/入度**:
    - 出度极高 (Out-degree) = 问题的根本源头 (Source)。
    - 入度极高 (In-degree) = 问题的核心症结 (Sink)。
- **专家提示**: "箭头指向最多的节点，是你要在报告中重点陈述的矛盾点。"

### 2.9 PDPC (过程决策程序图)
- **应用场景**: 风险管控。适用于“制定对策”阶段的预防预案设计。
- **DSL 特性**:
  - `start/step/end/countermeasure` 节点类型标识。
  - `id1--id2 [OK/NG]` 判定逻辑分支。
- **逻辑演化**: 动态路径模拟。如果 A 方案受阻，B 方案自动激活。
- **专家提示**: "PDPC 不是流程图，它是‘容错图’。它的价值在于 NG 分支上的 Plan B。"

### 2.10 矢线图 (Arrow Diagram / CPM)
- **应用场景**: 进度计划。适用于“实施对策”阶段的时间管控。
- **DSL 特性**:
  - `Event` 代表事件里程碑；`->` 实活动；`..>` 虚活动 (Dummy)。
  - `ShowCritical: true` 高度自动化定位关键路径。
- **深度算法**:
  - **CPM/PERT**: 基于最早开始/最晚结束时间计算总时差。
- **专家提示**: "虚活动（时长为0）是描述逻辑约束的唯一手段。关键路径上的延迟将导致整个项目的终结。"

### 2.11 矩阵项目图 (Matrix Plot)
- **应用场景**: 多元变量关联。适用于深层次的因子挖掘分析。
- **DSL 特性**:
  - `Mode: Matrix (自相关) / YvsX (多元影响)`。
  - `Smoother: Lowess` 专业回归算法。
- **分析策略**:
  - 观察对角线分布（确认采样是否偏置）。
  - 利用 Lowess 平滑线识别非线性回归规律。
- **专家提示**: "群体(Group)变色能识别分层数据带来的假相关。"

### 2.12 亲和图 (Affinity Diagram / KJ法)
- **应用场景**: 信息归纳。从混沌的感性信息中找规律，适用于“选题理由”或“原因初步筛选”。
- **DSL 特性**:
  - `Type: Card` 分组渲染；`Item` 递归定义。
- **逻辑模型**: 
  - **分层归类**: 想法 → 归组 → 提炼主题。
  - **思维洞察**: 每一个无法归类的异常点，都可能是创新的新维度。
- **专家提示**: "严禁预设分类。让数据自己‘说话’，通过‘亲和性’建立逻辑岛屿。"

## 3. Agent 开发范式 (Agent Paradox)

### 3.1 意图识别路径 (Intent Path)
1. **语义定位**: 分析用户 Prompt 中的关键词（如“不稳定”、“二八”、“关键路径”）。
2. **Schema 匹配**: 根据 `chart_spec.json` 触发对应的 `chart_grammar`。
3. **数据清洗**: 将杂乱文本抽提为 `- x: y` 或 `Rel: a -> b` 的标准语法。
4. **视觉增强**: 根据分析结果自动选择 `Color[Root]`（异常用红，稳定用蓝）。

### 3.2 严禁事项 (Constraints)
- 严禁在没有原始数据的情况下编造控制图 (SPC) 的控制限。
- 严禁在鱼骨图的同一层级混用 5M1E 与 4P 维度。
- 严禁忽略 `Title` 属性，这是 DSL 合法性的根基。

## 4. QC 成果报告书应用节点指南 (QC Report Guide)

| QC 阶段 | 推荐组件 | 逻辑锚点 | 视觉策略 |
| :--- | :--- | :--- | :--- |
| **P - 选题理由** | 饼图 (Basic), 亲和图 | 现状差距, 意见聚焦 | 简洁, 突出差异 |
| **P - 现状调查** | 排列图, 直方图 | 关键少数, 分布特征 | 红色标注 A 类因素 |
| **P - 分析原因** | 鱼骨图, 关联图, 矩阵图 | 因果逻辑, 系统映射 | 层级化, 逻辑线高亮 |
| **P - 原因确认** | 散点图, 矩阵项目图 | 统计显著性 | 显示回归线与 R 值 |
| **P - 制定对策** | PDPC, 矩阵图 | 预案分支, 决策打分 | 多分支, 对比分值 |
| **D - 实施对策** | 矢线图 (Arrow), 流程图 | 进度进度, 逻辑约束 | 关键路径高亮 |
| **C - 效果检查** | 控制图 (SPC), 雷达图 | 稳定性, 综合提升 | 3σ 线, 面积增长 |

## 5. 工程哲学与演进准则 (Engineering Philosophy & Evolution)

### 5.1 增补唯一定律 (The Law of Persistent Augmentation)
- **核心准则**: 针对 `DSL_SYNTAX_MANUAL.md` 与本宣示录的所有修订，必须严格遵循“只增不减”原则。
- **演进逻辑**: 历史语法是系统兼容性的基石。任何新功能的加入都应作为现有体系的超集，严禁删除、重命名或篡改已定义的有效指令，确保知识仓库的持续累积与长期可读性。

### 5.2 专家知识注入 (Expert Knowledge Injection)
- **透明化理念**: 对于外部 AI 客户端，全量语法及其背后的质量管理专家逻辑必须是透明且可感知的。
- **实现路径**: 代理人应通过自描述协议 (如 MCP 资源注入)，动态地将 DSL 细节与业务建议推送到 Prompt 上下文中，确保 AI 推理的完整性、场景适配性与渲染质量。
